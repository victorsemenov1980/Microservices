Микросервисы на Django и Flask

Только базовая логика. Адаптация базы данных в условиях Django и SQLAlchemy не занимает какого либо времени и труда, сами сервисы также могут быть абсолютно другими. Вопрос входа и выхода в основном стоит. 

Для запуска в каждом каталоге 
docker-compose up --build
Потом остановить и просто
Docker-compose up

Фронта нет совсем поэтому все api проверяются через postman (как самый удобный вариант)

Авторизация по JWT, издает Django, он же имеет админку, позволяет корректировать базу данных и прочее. 
Конкретно API Django авторизацией пока не защищено

Приложение Flask имеют свою базу данных, обновляет следом за Django, в обратную сторону тоже работает. 
Общение через Rabbit, а конкретно через CloudAMPQ

Верификация JWT напрямую через request. Хранение токенов в сессии в Redis на стороне Flask пока не реализовано. Оно нужно чтобы не было потока запросов.

Контейнеры делались на Mac
В приложении main обратите внимание на строчки http://docker.for.mac.localhost.....
